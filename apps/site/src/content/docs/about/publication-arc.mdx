Publication follows a **four‑act arc**.

The purpose of the arc is sequencing: publish concepts first, then publish implementation details later — so the implementation doesn’t swallow the practice.

## Outline

- **Prologue — A Living Practice of Clarity**
  - What the practice is, and why “trace” matters.
- **Act I — The Condition**
  - Acceleration without integration: pressure, rework, fragmentation.
- **Act II — The Discipline**
  - Trace over assertion: assumptions, boundaries, uncertainty, checks.
- **Act III — The Structure**
  - Federation: nodes and bridges, meaning without a centre.
- **Act IV — The Demonstration**
  - The repository is published. Source history and implementation details become visible.

## What is released when

- In **Acts I–III**, this site is the public reading surface.
- In **Act IV**, the repository is released as a demonstration node.

This is not “missing trace”. It is intentional sequencing.
