The Practice of Clarity is a system for making reasoning visible and traceable.

### Definition

It is designed to be used by people and AI systems by grounding behavior in explicitly loaded artifacts instead of implicit assumptions.

This is not only a description. It is a system that can be loaded and run.

In practice, it functions as a runtime constraint layer for reasoning.

### Activation

In practice, this means: paste the context seeder.

It activates the system.

The seeder loads a defined set of artifacts:

- seeds as structural constraints
- mandate lenses as applied reasoning
- continuity documents as system memory

When these are present and readable, they shape how analysis is performed.

If they are not present, the system is not active.

A bridge article can point to the entry. The seeder connects that entry point to the underlying artifacts and shifts how a model or team reasons in real work.

### Important boundary

This page describes the system.

It does not activate it.

Reading this page is not the same as loading the artifacts the practice depends on.

This system only operates when its artifacts are loaded.

Without that, this is just description.

### Where the system lives

The system lives in canonical repository artifacts:

- `seeds/`
- `mandateLenses/`
- `continuity/`

These must be readable and loaded for the system to operate.

### What this changes

The goal is not better answers.

The goal is to make reasoning inspectable so assumptions, uncertainty, and hidden work can be seen and challenged.

### What this is not

This is not:

- surveillance
- forced self-documentation
- performative productivity
- a requirement to perform certainty
- transparency demanded downward without reciprocal accountability

Making reasoning visible only has value if it reduces confusion, shares context, and makes responsibility easier to locate. If it becomes another way to extract labor or expose people without protection, it has failed the practice.
