This is where the stable reasoning contract moves toward use.

A mandate lens takes what the seeds keep steady and shapes it toward a specific
kind of pressure.

It helps when the practice already makes sense and you want support bringing it
into real work without turning it into a generic framework or process layer.

A mandate lens makes three things easier to see:

- what pressure the lens is meant to make visible
- which lens can be tried right now
- where the source texts live when you want to inspect the contract behind it

### Reading and trying

Reading about a lens is not the same as loading it into a live reasoning environment.

These pages help you recognize what a lens is for.

Trying one begins when the relevant source context is brought into a model or
agent that can follow extended guidance and sustain a longer reasoning task.

That source context is not only the seeder. It relies on the seed contract
behind the practice, the lens text, and the context seeder that carries the
relevant parts of that contract into use.

### Current lens

The first public starting point is [Sensible Defaults](/en-us/core-system/mandate-lenses/sensible-defaults-a-lens-you-can-load/).

It is usable now, but it is not presented as finished. It is the first public
lens surface for LLM-assisted coding where plausible output can move faster than
the reasoning needed to inspect, review, and safely integrate the work.

### Current boundary

There is only one public lens starting point right now.

That is enough. The public surface does not need to pretend it is already a
large catalog, a finished method library, or a complete set of mandate lenses.
This layer is just beginning, and the first public surface should stay
inspectable, usable, and open to revision.
