The earlier Acts explain why reasoning needs to stay visible under AI speed.

Act IV shows how the public site and repository make one public node
inspectable. `Sensible Defaults` is the first usable runtime guidance surface in
this public node for LLM-assisted coding.

This is not a framework you learn.

It is a lens you can load as source context into a live reasoning environment.

It is usable now, but it is not presented as finished. This public entry helps
you understand what the lens is for, what kind of model or agent it requires,
and how the source context starts shaping reasoning during work.

That does not make the lens finished. It makes this first usable surface visible
enough to inspect, try, and revise.

### Opening pressure

AI-assisted coding often breaks before implementation visibly fails.

It breaks when the generated plan, code, or summary looks clean but the real
work is not visible:

- hidden coordination
- partial understanding
- planning based on generation time instead of integration time
- risk pushed to the end
- cognitive overload carried by the people who have to make generated work actually hold

### What the lens is for

This lens is for people coding with AI when output is moving faster than the
reasoning needed to understand, review, and safely integrate the work.

It is especially useful when an AI-generated plan, implementation, or summary
looks plausible but the real shape of the work is still compressed: data,
infrastructure, tests, QA, security, deployment, ownership, or product impact
may not yet be visible.

This lens only starts working when its source context is brought into a live
LLM reasoning environment with a model or agent capable of following extended
guidance and sustaining a longer reasoning task.

That context is not only the seeder. It relies on the reasoning-contract seeds
behind the practice, the lens text, and the context seeder that carries relevant
source context into use.

This public node exposes the fuller PoC source context so the lens can be
inspected.

That is not the same as saying every project should load the whole PoC posture.
When trying the move elsewhere, use the smallest relevant slice: the local
pressure, the boundary the agent should respect, the evidence it should inspect,
and the checks that make confidence honest.

Until a smaller universal seed pack exists, treat the PoC source surfaces as an
inspectable reference and adapt them carefully to the project at hand.

For now, use the linked source surfaces below as the public path to the source
context. Without that source context, this page remains description.

It is especially useful in:

- AI-assisted coding
- legacy codebases
- chaotic planning
- overloaded delivery environments
- unclear implementation boundaries
- work where plausible output arrives faster than people can inspect and integrate it

### What happens when it loads

When the lens is active:

- assumptions are surfaced instead of hidden
- observation is separated from inference
- active and missing delivery domains become visible
- false certainty is reduced by forcing the next check into view

If you want to try it in another project, do not load the whole set by default.

If the source context is not loaded, the lens is not active.
