This is not a framework you learn.

It is a lens you load.

`Sensible Defaults` is the first bridge from reading into use.

This public entry helps you understand what the lens is for, then try it if it fits your situation.

### Opening pressure

Modern delivery work often breaks down before implementation fails.

It breaks when the plan looks clean but the real work is not visible:

- hidden coordination
- partial understanding
- planning based on coding time only
- risk pushed to the end
- cognitive overload carried by the people who have to make things actually hold

### What the lens is for

This lens is for delivery contexts where output is moving faster than shared understanding and the real shape of the work is being compressed.

For example, if a plan lists features but says nothing about data, infrastructure, or QA, the lens flags those missing delivery domains before the work drifts.

This lens only starts working when its text is brought into the conversation.

In practice, that means opening `context-seeder.md` and pasting it into the model or agent you are using, or loading the same file from a local checkout.

If you are using tools like ChatGPT, Claude, or similar systems, bring the seeder text into that conversation.

Without that, the lens remains description.

It is especially useful in:

- legacy codebases
- chaotic planning
- overloaded delivery environments
- AI-assisted work where plausible output arrives faster than teams can integrate it

### What happens when it loads

When the lens is active:

- assumptions are surfaced instead of hidden
- observation is separated from inference
- active and missing delivery domains become visible
- false certainty is reduced by forcing the next check into view

### Try it

If you want to use the lens directly, start with:

If the source cannot be loaded, the lens is not active.
