This is where the Acts move from explaining the practice to showing how it can be inspected.

Up to this point, the Acts named the condition, introduced the discipline, and showed how the practice can move between contexts without losing visible reasoning.

Now the question changes: what does it look like when one bounded trace surface is public and in use?

### The shift

A public node is not proof by itself. It is a bounded trace surface where parts
of the work remain visible enough to inspect.

Here, the public node is the public site and repository together.

It can show:

- what is claimed on the public site
- which routes and register surfaces carry those claims
- which source surfaces, source context, and reasoning contracts support the work
- what changed over time
- what failed, moved, or remained unresolved

A finished page can hide how it was made. A public node makes enough of the
support visible to inspect: what held, what changed, what failed, and what
still needs checking.

### What inspection makes possible

Inspection means structural access.

It allows a reader to:

- compare claims with their underlying trace
- see assumptions before they stabilize into language
- follow revisions across time
- detect where clarity holds and where it collapses

Without inspection, the practice risks becoming vocabulary. With inspection, it becomes accountable in practice.

### Guided inspection

A public node should be readable in layers.

Start with the public claim. Then inspect the surface that should support it.

Here, that can mean:

- read the public page where the claim appears
- compare it with the route and register surfaces that carry it
- inspect the [repository](https://github.com/Mikeys-Tech-Lab/poc) when the claim needs a deeper trace
- use the [repository architecture](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/architecture/workspace.md) for deeper build detail
- use the [Evolution Arc](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/guidance/evolution-arc.md) to follow how the work changed over time

These surfaces do not replace the Acts. They show how the practice is held,
revised, and exposed to challenge under real work.

### What this node reveals

A real node does not present itself as finished or complete.

It contains:

- public surfaces that show what is currently claimed
- repository surfaces that show how decisions and structure evolved
- revision history that shows how meaning changed
- reasoning contracts that state what should remain stable
- explicit boundaries where claims stop
- uncertainty that remained visible before the public page was finalized

### Seeds, repository, and practice

Seeds are reasoning-contract texts. They help the work keep its shape as it
moves.

They are not replaced by the public node.

The public node shows how those contracts are carried under real conditions. It
is where the practice becomes inspectable. `Sensible Defaults` is the first
usable runtime guidance surface in this public node, built for LLM-assisted
coding and usable now but not finished.
