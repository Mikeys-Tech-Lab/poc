If the practice is useful, it will move between contexts.

The moment it moves, interpretation begins. Under delivery pressure, this
accelerates. People reuse language to move quickly. Meaning can compress while
the words stay familiar.

So the question is no longer only what the practice is.

The question becomes: how does the practice stay legible when more than one
bounded trace surface carries it?

<ReadingBeat>
  The problem is not movement. The problem is drift that becomes invisible while
  the language still looks familiar.
</ReadingBeat>

### Nodes

A node is a bounded trace surface where the practice is carried in real work
with visible reasoning, explicit limits, and inspectable change.

A node is not created by using the language of the practice. It is created when
the work leaves enough trace for others to see what was claimed, what changed,
what supported it, and where the limits were.

A team, project, or community does not become a node by name alone. It can only
host a node if the practice is carried there through a surface that can be
inspected.

### Bridges

When more than one node exists, comparison becomes possible.

A bridge makes it possible to connect or contrast how the practice is carried
across bounded trace surfaces.

It may be:

- a shared artifact
- a side-by-side comparison
- a trace of how two bounded contexts approached the same problem

The purpose is not agreement. It is orientation.

### Drift

Drift is not a failure by itself. It is what happens when wording travels faster
than the reasoning contracts, public surfaces, and traces that ground it.

Drift begins when:

- language is copied without the conditions that gave it meaning
- examples are repeated without their constraints
- outputs are shared without the reasoning that produced them

The problem is not that drift exists, but when it becomes invisible.

### Working with drift

Drift cannot be eliminated. It can only be made visible.

That shifts the response:

- not enforcement
- not correction from a center
- but clarification through visible traces and texts

Instead of arguing definitions alone, nodes can show what they did, what they
assumed, what changed, and what remains unresolved.
