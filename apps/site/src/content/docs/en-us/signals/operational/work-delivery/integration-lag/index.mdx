Integration Lag is a series about one delivery condition: output can move faster
than the shared understanding it depends on.

The series names that condition, then stays with it. It does not treat speed as
the problem or AI as the cause. It follows what happens when work ships before
people have had time to integrate what changed.

### What the series covers

The three signals build on each other:

1. [We Started Shipping Faster. We Also Started Understanding Less.](/en-us/signals/operational/work-delivery/integration-lag/we-started-shipping-faster-understanding-less/) names integration lag and shows how AI can make unresolved uncertainty look finished.
2. [A Path Through Integration Lag](/en-us/signals/operational/work-delivery/integration-lag/a-path-through-integration-lag/) moves from naming the gap to keeping understanding attached to the work, without turning it into a framework rollout.
3. [The Verification Tax](/en-us/signals/operational/work-delivery/integration-lag/the-verification-tax/) stays with the checking, correcting, and context work that remains once drafts arrive faster than grounded confidence.

Read them in order for the full arc, or start with the signal that matches the
pressure you already recognize.
