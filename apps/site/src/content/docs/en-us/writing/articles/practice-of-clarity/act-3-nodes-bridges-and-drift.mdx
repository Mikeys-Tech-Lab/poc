If the practice is useful, it will travel.

The moment it travels, interpretation begins. Under delivery pressure, this happens faster. Teams reuse language to move quickly. Meaning compresses. The surface remains.

So the question is no longer only what the practice is.

The question becomes: how does the practice stay legible when more than one node is using it?

### Nodes

A node is a place where the practice is actually carried in real work.

Not described. Not advertised. Carried.

A team, a project, a community. A node is defined by what it does under pressure, not by how it describes itself.

### Bridges

When more than one node exists, comparison becomes possible.

A bridge is a visible way to connect or contrast how the practice is applied across nodes.

It may be:

- a shared artifact
- a side-by-side comparison
- a trace of how two nodes approached the same problem

The purpose is not agreement. The purpose is orientation.

### Drift

Drift is not a failure. It is what happens when wording travels faster than the artifacts that ground it.

Drift begins when:

- language is copied without the conditions that gave it meaning
- examples are repeated without their constraints
- outputs are shared without the reasoning that produced them

The problem is not that drift exists. The problem is when drift becomes invisible.

### Working with drift

Drift cannot be eliminated. It can only be made visible.

That changes the response:

- not enforcement
- not correction from a center
- but clarification through artifacts

Instead of arguing definitions, nodes can show what they did, what they assumed, and what they did not resolve.

### Refusal of association

Not every use of the language carries the practice.

If reasoning is not made visible, the practice is not being carried.

That boundary is not punishment. It is what keeps the practice from being reduced to language alone.

### Forward edge

Once nodes, bridges, and drift are visible, the next question is what happens when a node becomes publicly inspectable.
