Mandate lenses are the applied package family for the Practice of Clarity.

They are not the seed layer and they are not continuity.

They are the context-facing packages that carry the practice into real work.

### What this section does

This section is the public bridge into those packages.

It helps a reader understand:

- what a mandate lens is for
- which lenses exist on the live surface
- where the runtime source of truth actually lives

### Reading vs activation

Reading this section is not the same as loading a lens.

Bridge pages orient the reader.

Operational grounding begins when the canonical package artifacts are loaded from
the repository.

### Package shape

Each mandate lens package contains:

- `README.md` - local entry and lineage notes
- `lens.md` - canonical application artifact
- `context-seeder.md` - derivative portable runtime entry

See the root family package:

- [mandateLenses/README.md](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/README.md)

### Current live lens

The first live lens is [Sensible Defaults](/en-us/writing/articles/practice-of-clarity/sensible-defaults-a-lens-you-can-load/).

Its canonical repository surface is:

- [SensibleDefaults/README.md](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/SensibleDefaults/README.md)
- [SensibleDefaults/lens.md](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/SensibleDefaults/lens.md)
- [SensibleDefaults/context-seeder.md](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/SensibleDefaults/context-seeder.md)

### Current boundary

There is only one public lens right now.

That is enough to make the family visible without pretending it is already a
large catalog.
