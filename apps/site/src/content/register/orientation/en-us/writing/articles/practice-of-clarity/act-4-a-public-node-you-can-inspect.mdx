This is where the work stops only describing the practice and starts showing it in use.

Until now, the articles have explained the ideas. Here, the repository becomes part of the public explanation.

<ReadingBeat>
  Inspection changes the practice because it lets people see the trace, not just the finished claims.
</ReadingBeat>

### What this changes

Instead of only reading finished text, you can inspect:

- how the current structure and decisions came to be
- changes over time
- what was uncertain when decisions were made
- what was not fully resolved

That matters because it shows not just what was decided, but how those decisions were formed and where they are still open.

### What the repository is

The repository is a working example of the practice in use.

It is not the whole work. It is a visible, bounded node.

If you want to look more closely, the repository includes a [manual reading path](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/onboarding/manual.md) and an [Evolution Arc](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/onboarding/evolution-arc.md) for following how the work changed over time.

You can inspect the public node directly in the [repository](https://github.com/Mikeys-Tech-Lab/poc).
