This is not another Act.

It is the first bridge from reading the practice to using it.

If your work feels confusing, overloaded, or harder to plan than it should, this lens helps make the hidden parts visible.

For example, if a plan names features but says nothing about data, infrastructure, or QA, this lens helps surface that missing work early.

### What this lens is for

Use it when the work already feels busier, riskier, or more compressed than the
plan admits.

It helps you slow down just enough to see what the plan is carrying, what it is
leaving out, and what needs checking next.

This lens only starts helping when its text is actually brought into the
conversation.

In practice, that usually means opening the seeder link and copying the text
into the model or agent you are using, or loading the same text from a local
checkout.

If you are using tools like ChatGPT, Claude, or something similar, this is the
place where you paste that text.

Without that, this page is only description.

### What changes when the lens loads

- hidden work becomes easier to name
- missing delivery domains become easier to notice
- planning pressure becomes easier to inspect
- the next honest check becomes easier to ask for

This is not extra process. It is a way to reduce hidden load by making the work easier to see.

### Try it

If you want to use the lens directly, start here:

- [Seeder text](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/SensibleDefaults/context-seeder.md)
- [Lens text](https://github.com/Mikeys-Tech-Lab/poc/blob/main/mandateLenses/SensibleDefaults/lens.md)

Those linked texts are the fuller source if you want to try it in your own
work.

If the source cannot be loaded, the lens is not active.
