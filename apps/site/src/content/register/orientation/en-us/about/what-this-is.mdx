The Practice of Clarity is a system for making reasoning visible and traceable.

This is not only a description. It is a system that can be loaded and run.

### Short answer

It is designed to be used by people and AI systems by grounding behavior in explicitly loaded artifacts instead of hidden assumptions.

### How it works

It works by loading a defined set of artifacts:

- seeds for structural constraints
- mandate lenses for applied reasoning
- continuity documents for system memory and architecture

When these are present and readable, they shape how reasoning happens in practice.

If they are not present, the system is not active.

### How it is practiced

In practice, this means: paste the context seeder.

It activates the system.

A bridge article can help someone enter the system. The seeder loads the underlying artifacts and shifts how a model or team reasons in real work.

### Important boundary

This page describes the system.

It does not activate it.

This system only operates when its artifacts are loaded.

Without that, this is just description.

### Where the system lives

The system lives in canonical repository artifacts:

- `seeds/`
- `mandateLenses/`
- `continuity/`

These must be readable and loaded for the system to operate.

### What this changes

The goal is not better answers.

The goal is to make reasoning inspectable so assumptions, uncertainty, and hidden work can be seen and challenged.

### What this is not

This is not:

- surveillance
- forced reporting
- productivity theater
- pressure to sound certain when you are not
- one-way transparency that exposes people without protection

Making reasoning visible should reduce confusion and help people share context. If it becomes another burden or another way to control people, it stops being this practice.
