This is where the work stops only describing the practice and starts showing it in use.

Until now, the Acts have explained the ideas. Here, one bounded trace surface becomes publicly inspectable.

<ReadingBeat>
  Inspection changes the practice because it lets people see the trace, not just the finished claims.
</ReadingBeat>

### What this changes

Instead of only reading finished text, you can inspect:

- how the current structure and decisions came to be
- changes over time
- what was uncertain when decisions were made
- what was not fully resolved

That matters because it shows not just what was decided, but how those decisions were formed and where they are still open.

### What the public node is

Here, the public node is the public site and repository together.

The site is where most readers enter the work. The repository keeps more of the structure, history, source surfaces, and supporting material inspectable.

It is not the whole work. It is one visible, bounded trace surface.

If you want to look more closely, the repository includes a [workspace architecture page](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/architecture/workspace.md) and an [Evolution Arc page](https://github.com/Mikeys-Tech-Lab/poc/blob/main/docs/guidance/evolution-arc.md) for following how the work changed over time.

You can inspect the deeper trace directly in the [repository](https://github.com/Mikeys-Tech-Lab/poc).
