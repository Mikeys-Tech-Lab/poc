This is where the practice moves toward use in real situations.

A [mandate lens](/en-us/about/glossary/#mandate-lenses) helps when the practice
makes sense, but you need support bringing it into a specific kind of pressure.

A mandate lens makes three things easier to see:

- what the lens makes visible in practice
- which usable surface exists right now
- where the deeper source text lives when you want to inspect it

### Important boundary

These pages introduce a lens.

They do not activate it by themselves.

Trying a lens begins when the relevant [source context](/en-us/about/glossary/#source-context)
is brought into a model or agent that can follow extended guidance and sustain a
longer reasoning task.

That source context is not only the seeder. It relies on the reasoning-contract
seeds behind the practice, the lens text, and the context seeder that carries
relevant source context into use.

That does not mean every project needs the whole PoC source posture. This public
node shows the fuller inspectable context. When trying a lens elsewhere, bring
only the relevant slice and pair it with the local project's own source context.

### Current lens

The first usable runtime guidance surface in this public node is [Sensible Defaults](/en-us/core-system/mandate-lenses/sensible-defaults-a-lens-you-can-load/).

It is usable now, but it is not presented as finished. It is built for
LLM-assisted coding contexts where plausible output can move faster than the
reasoning needed to inspect, review, and safely integrate the work.

### Current boundary

There is only one usable runtime guidance surface in this public node right now.

That is enough. This public node does not need to pretend it already contains a
large catalog, a finished method library, or a complete set of mandate lenses.
This layer is just beginning, and the first usable surface should stay
inspectable, usable, and open to revision.
