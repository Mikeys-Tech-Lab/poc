## What the lens is for

`Sensible Defaults` is a delivery-realism lens.

It exists for situations where implementation work is visible, but the rest of
the delivery load is not.

It helps surface:

- hidden delivery domains
- compressed planning assumptions
- invisible coordination work
- optimism that is standing in for a real check

The point is not to make planning heavier.

The point is to make hidden work visible early enough to reduce pressure and
rework.

---

## What it does not do

This lens is not:

- a scoring system
- a compliance gate
- a performance tool
- a reason to demand exhaustive paperwork

If it adds pressure instead of releasing it, it has drifted.

1. Read this page if you need the framing first.
2. Load `mandateLenses/SensibleDefaults/lens.md` if you need the canonical lens.
3. Load `mandateLenses/SensibleDefaults/context-seeder.md` if you need the
   portable runtime entry.
4. Confirm that the full seed set in `seeds/` is readable before claiming that
   the lens is active.

Operational grounding begins when the canonical lens and required seeds are
actually loaded.

Reading this page alone does not activate the lens.

## Small summary of the lens contract

When explicitly invoked, the lens asks for:

1. **Active domains**
2. **Missing domains**
3. **Often unplanned work**
4. **Risk concentrations**
5. **Timeline pressure points**
6. **Suggested next questions**

Keep the output concise unless the user asks for depth.
